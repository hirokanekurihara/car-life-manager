# カーライフマネージャー（個人向け車管理Webアプリ）

## プロジェクト名
カーライフマネージャー

## ゴール
- 個人利用向けに、**最大5台の車両**を完全フロントエンドで一元管理
- iPhoneで使いやすいUI/UX（ボトムナビ、レスポンシブ）
- デスクトップでも使いやすいUI（サイドナビ、2カラム）
- PWA化によるホーム画面追加・アプリライク動作
- IndexedDB保存＋Service Workerキャッシュで**完全オフライン動作**
- 画像を含むデータの**完全バックアップ/復元（ZIP）**

---

## 現在実装済み機能（Completed）

### 1) 基本基盤
- React（UMD）+ Tailwind CSS（ローカル配置）でSPA構築
- Dexie.js（IndexedDB）によるローカル永続化
- Chart.jsによる燃費推移グラフ + 費用分析グラフ（月別/カテゴリ別/車両別）
- JSZipによるバックアップZIP作成/復元
- PWA対応（`manifest.json`, `sw.js`, icons）
- オフラインキャッシュ（アプリ本体アセット）
- iPhoneホーム画面起動向けのセーフエリア最適化（`safe-area-inset-top/bottom` + `100dvh`）
- UIリフレッシュ（アプリ風デザイン）: グラデーション背景、ガラス調カード、強化ナビゲーション、モバイル下部ナビの強調表示
- iOS風UI強化: 角丸を深めたパネル、タブ切替アニメーション、アイコンバッジ付きナビゲーション（PC/モバイル共通）

### 2) 車両管理（サマリー表示）
- ホーム画面は車両サマリー専用（カード表示のみ）
- ホーム画面を車両カード縦並びで表示（カード全体タップで `車両情報` タブへ遷移）
- カード上部に車両写真（未登録時は車種名キーワードで分類したシルエット：軽=水色/セダン=青/SUV=緑/ミニバン=紫/その他=灰）
- カード中央に車種名・ナンバー・現在走行距離を大きめフォントで表示
- カード下部に期限表示（車検/保険/税）を「〜まで○日」で表示し、7日以内=赤、8-30日=オレンジ、31-60日=黄、61日以上=緑で色分け
- **車両情報タブ（単独）**に、選択車両の入力済み項目のみを一覧表示
- 車両情報タブに、保険タブで登録した保険契約（自賠責/任意）を連動表示
- 車両情報タブ下部に選択車両のメンテナンス履歴を時系列で表示
- **車両登録タブ（単独）**で新規登録・登録済み車両の編集/削除を実施
- 車両登録フォームは折りたたみ式（初期は閉じた状態、見出しタップで開閉）
- 車両登録フォームから保険項目を分離（保険は保険タブで管理）

### 3) メンテナンス・予約（重要機能）
- ステータス3段階：未予約 / 予約済み / 完了
- 予約情報：予約日、予約時間、店舗、担当者、電話（`tel:`リンク）
- 店舗リスト管理（店舗名/電話番号/担当者）を追加し、整備登録フォームで選択すると該当項目を自動入力
- Googleカレンダー自動登録チェックボックス（整備保存時にONの場合、Googleカレンダー作成画面を自動起動）
- 予約詳細カードから**Google Calendarに追加**（Googleブルーのカレンダーボタン）
- 実施記録：実施日、走行距離、作業内容、費用、次回予定日
- 画像添付：最大5枚、Canvas圧縮（最大800px / 品質0.7）
- カテゴリ：初期カテゴリ + 追加/編集/削除（`NOJメンテナンス`を標準カテゴリとして保持）
- `NOJメンテナンス`選択時のみ、現在チケット残数を表示
- NOJチケット運用：
  - メンテ実施（利用）で **1枚減算**（ステータス`完了`時のみ）
  - チケット購入で入力枚数を **加算**
  - 残数不足時は利用をブロックし、購入への切替をガイド
- 検索・絞り込み・並び替え（ステータス/車両/キーワード/期間指定/日付・費用・距離）
- 期間プリセット（今月 / 3か月 / 6か月 / 1年）
- 保存済み検索条件（ローカル保存・呼び出し・削除、並び順とタブ状態も保存）
- 一括操作（複数選択・一括ステータス変更・一括削除）
- 一括編集（カテゴリ/ステータス/店舗/担当/電話/次回予定日を選択レコードへ同時適用）
- 各記録のインライン編集フォーム（主要登録項目を編集可能）/ステータス更新/削除

### 4) 燃費・給油記録
- 給油日、オドメーター、給油量、単価、総額、スタンド名
- **レシートOCR自動入力（Tesseract.js / 完全クライアントサイド）**
  - 「📷 レシートを撮影して自動入力」ボタン（iPhoneカメラ起動）
  - 前処理（横幅1000pxリサイズ / JPEG圧縮 / グレースケール + コントラスト補正）
  - 抽出項目：給油日時、給油量(L)、単価(円/L)、合計金額、スタンド名
  - 読み取り中インジケーター、確認画面（この内容で保存 / 手動で修正 / 再撮影）
  - 完全失敗時は手動入力モードへ自動遷移、部分成功時は抽出項目だけ自動反映
  - OCR成功時のレシート画像をIndexedDBへ保存し、給油記録詳細で表示・削除可能
- 前回給油から区間距離・実燃費(km/L)自動算出
- 折れ線グラフで燃費推移を表示
- ガソリン単価グラフ（フィルタ連動）
- ガソリン料金グラフ（フィルタ連動）
- 月間集計（ガソリン代・平均燃費）
- 期間プリセット + 保存済み検索条件（並び順とタブ状態を含む）
- 並び替え（日付/金額/燃費）
- 記録の編集/削除

### 5) タイヤ専用管理
- 夏/冬タイヤ区別
- 履き替え・ローテーション・交換・残り溝点検記録
- 残り溝1.6mm以下で交換警告表示
- 履き替え実施時に「現在装着タイヤ」を自動更新
- 履歴の編集/削除

### 6) 自由メモ・日記
- 車両別メモ（日付・タグ・本文）
- 写真添付（圧縮保存）
- 重要フラグ（★）を上部固定表示
- キーワード検索
- メモ編集・重要切替・削除

### 7) 年間カレンダービュー
- 月切替可能なカレンダー
- 色分けイベント表示
  - 車検期限（赤）
  - 予約（青）
  - 完了整備（緑）
  - 給油/税関連（黄系）
- 日付タップで当日イベント詳細表示
- **カレンダーをエクスポート（.ics）** ボタンを配置（カレンダー画面）

### 8) 保険タブ（新設）・緊急連絡先
- **保険専用タブ**を追加（自賠責 / 任意保険を分離管理）
- 保険登録項目：対象車両、保険種別、保険会社、代理店、各電話番号、開始日、満了日、証券番号、メモ
- 保険会社/代理店リスト（マスタ）を作成し、選択で会社名/代理店名と電話番号を自動入力
- 任意保険は**補償内容画像**を添付保存し、一覧で参照可能
- 保険レコード一覧表示（車両、契約期間、会社/代理店、電話、証券番号）
- 登録済み保険レコードのインライン編集（契約内容更新）に対応
- 緊急連絡先一覧（初期データあり）
- 名前/電話番号編集
- すべて`tel:`リンクで1タップ発信

### 9) 強化アラート・期限管理
- 監視対象：車検満了・保険満了・税期限
- 色分けアラート（7/30/60日閾値）
- ホーム上部に緊急アラート表示
- 予約3日前以内のリマインダー表示
- 期限管理カード（車検満了日/保険満了日/自動車税納付期限）から**Google Calendarに追加**

### 10) 費用分析
- 月別総維持費（燃料費＋メンテ費）バーグラフ
- カテゴリ別メンテ費内訳ドーナツグラフ
- 車両別総費用比較グラフ
- 分析期間切替（今月 / 3か月 / 6か月 / 1年）
- ヘッダーの対象車両選択に連動した分析表示（比較グラフは全車両）

### 11) 画像圧縮設定
- 圧縮品質を選択（0.5 / 0.7 / 0.85）
- 最大辺を選択（600 / 800 / 1200 px）
- 整備画像の1件あたり上限枚数を選択（1〜5枚）
- 既存画像の一括再圧縮（車両写真/整備画像/メモ写真）
- 再圧縮対象の選択（全画像 / 車両写真のみ / 整備画像のみ / メモ写真のみ）
- 再圧縮の進捗バー表示（処理枚数 / 総枚数 / 進捗%）
- 推定残り時間の表示（処理速度ベース）
- 再圧縮中の一時停止 / 再開
- 設定はローカル保存され、登録/編集時の画像圧縮へ反映

### 12) バックアップ・復元（重要）
- 全データをZIPで出力（画像をBase64化して含有）
- ZIPから全データ復元（画像Blob再構築）
- CSVエクスポート（全テーブルをCSV化してZIP出力）
- 月1回バックアップ推奨通知（前回日時ベース）
- **手動クラウド同期**（`appSyncSnapshots` テーブル）
  - 「この端末のデータをクラウドへ保存」
  - 「クラウドのデータをこの端末へ反映」
  - 端末名を付与してPC/iPhone間で同一データを移行

### 13) カレンダー連携（Google Calendar / iCalendar）
- Google Calendar ワンタップ登録
  - 予約詳細画面・期限管理画面に `Google Calendarに追加` ボタンを配置
  - URL: `https://calendar.google.com/calendar/render?action=TEMPLATE`
  - パラメータ: `text`, `dates`, `details`, `location`, `ctz=Asia/Tokyo`
- iCalendar（`.ics`）エクスポート
  - 配置: 年間カレンダー画面 / その他（設定）画面
  - 対象: 予約（予約済み・完了）/ 車検満了日（30日前アラーム）/ 保険満了日（60日前）/ 自動車税納付期限（30日前）
  - 選択範囲: `すべて` / `予約のみ` / `期限のみ` / `車両別`
  - ファイル名: `車管理_YYYY-MM-DD_[選択範囲].ics`
- タイムゾーン対応
  - `VTIMEZONE` に `Asia/Tokyo`（JST, UTC+9）を付与
  - 時刻イベントは `DTSTART;TZID=Asia/Tokyo` / `DTEND;TZID=Asia/Tokyo` で出力

#### カレンダー連携の使い方（日本語ガイド）
1. **整備タブ**で対象の予約カードを開き、`Google Calendarに追加` を押す
2. **車両情報タブ**の期限情報を確認しつつ、必要に応じてGoogleカレンダー登録を実施
3. **カレンダータブ**または**その他タブ**で `カレンダーをエクスポート（.ics）` を押す
4. エクスポート対象（すべて/予約のみ/期限のみ/車両別）を選んで `.ics` を保存
5. `.ics` をGoogle/Apple/Outlookへ取り込むと、JST時刻で予定が反映される

- 出力: `car-manager-backup-YYYY-MM-DD.zip`

- CSV出力: `car-manager-csv-YYYY-MM-DD.zip`（`vehicles.csv`等を同梱）
- iCalendar出力: `車管理_YYYY-MM-DD_[選択範囲].ics`

## データモデル / 構造 / 保存サービス

### 保存先
- ブラウザ `IndexedDB`（Dexie）

### テーブル
- `vehicles`
  - 基本情報、消耗品スペック、期限、写真Blob、税情報、現在装着タイヤ、NOJチケット残数（`nojTicketRemaining`）
  - 保険の主データは `insurances` テーブルで管理（連動表示用に一部要約情報を保持する場合あり）
- `maintenances`
  - 予約・完了情報、費用、走行距離、カテゴリ、領収書画像Blob配列、Googleカレンダー自動登録フラグ、店舗リスト参照ID
  - NOJチケット履歴（`nojTicketAction`, `nojTicketChange`, `nojTicketRemainingAfter`）
- `fuelLogs`
  - 給油記録、区間距離、実燃費、レシート画像Blob、OCR生テキスト
- `tireLogs`
  - 履き替え/ローテーション/交換/残り溝点検履歴
- `memos`
  - 車両別メモ、タグ、重要フラグ、写真Blob
- `categories`
  - メンテカテゴリ
- `contacts`
  - 緊急連絡先
- `shops`
  - 整備店舗マスタ（店舗名/電話/担当者）
- `insuranceContacts`
  - 保険会社/代理店マスタ（役割、名称、電話）
- `insurances`
  - 保険契約（車両ID、種別、会社/代理店、電話、開始日、満了日、証券番号、メモ、任意保険画像Blob）
- `settings`（予約）
- `appSyncSnapshots`
  - 端末間同期用スナップショット（`id=default`, `payload`, `device`, `updatedAt`, `version`）

## 補足
このプロジェクトはサーバー不要の静的Webアプリとして設計されています。データは端末ブラウザに保存されるため、機種変更時は必ずバックアップZIPを作成してから移行してください。
